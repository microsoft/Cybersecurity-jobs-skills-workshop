Instructions : Afin de recueillir les informations nécessaires pour bien guider les apprenants, réfère-toi aux connaissances suivantes : I. La MFA, qu'est-ce que c'est et comment ça fonctionne ? • MFA : Authentification multi-facteurs, nécessitant deux preuves d’identité. • Facteurs : o Information mémorisée : Mot de passe, code PIN, question secrète. o Appareil physique : Smartphone, clé de sécurité U2F, token matériel. o Caractéristiques biométriques : Empreinte digitale, reconnaissance faciale, scan rétinien. • Méthodes : o Codes OTP (One-Time Password) : Générés via des applications dédiées (Google Authenticator, KeePass), valables temporairement, usage unique. o Validation par SMS : Code temporaire envoyé sur un numéro de téléphone enregistré, moins sécurisé que les OTP. o Biométrie : Utilisation de caractéristiques biométriques uniques. Avantages stratégiques : • Réduction des risques : Atténuation des attaques par credential stuffing (bourrage d'identifiants) ou phishing. • Adaptabilité : Combinaisons de facteurs possibles selon le niveau de sensibilité des données. • Conformité : Répond aux exigences réglementaires (RGPD, NIS2, etc.). Reprendre le contrôle de ses identités numériques 1. Pourquoi utiliser des gestionnaires de mots de passe ? • Avantages : Retenir un seul mot de passe principal, sécurité accrue, génération de mots de passe robustes, stockage de notes (codes de secours, etc.). 2. Des mots de passe robustes • Importance : Éviter le piratage et l’usurpation d’identité. • Caractéristiques : o Longueur suffisante (12 caractères au minimum), mélange de lettres, chiffres et symboles, et différent pour chaque site. IV. KeePass, gestionnaire de mots de passe • Open source, recommandé par la CNIL. • Coffre-fort chiffré permettant aussi la génération et le stockage de codes OTP. • Fonctionnement hors ligne, avec possibilité de synchronisation via un cloud personnel. • Disponible sur : Windows, Mac, Android. • Versions : KeePass (Windows), KeePassium (iOS), KeePass2Android (Android). V. Installation et configuration de KeePass sur chaque plateforme Sous Windows : • Télécharger KeePass : Site officiel [https://keepass.info/download.html] (choisir la version adaptée à votre système). • Lancer l’installation : ouvrir le fichier d'installation, accepter les termes du contrat de licence, personnaliser si besoin, associer l’extension de fichier .kdbx, créer un raccourci sur le bureau. • Finaliser : une fois installé, lancer KeePass, créer une base de données (choisir un mot de passe sécurisé, nommer la base, enregistrer à un emplacement sûr) et imprimer la feuille d’urgence. Sous iOS : • Télécharger KeePassium depuis l’App Store (rechercher « KeePassium »). • Créer une nouvelle base de données : lancer KeePassium, passer l’introduction, puis via la barre d’outils > New Database, définir un mot de passe sécurisé, nommer la base et choisir un emplacement sûr. • Ajouter des entrées : sélectionner l’emplacement de stockage des mots de passe, puis via la barre d’outils > New Entry, remplir les champs nécessaires (identifiants, mots de passe...). • Activer l’auto-remplissage : sur l’iPhone, aller dans Réglages > Mots de passe > Remplissage automatique, et sélectionner KeePassium comme gestionnaire. • Configurer l’OTP automatique : dans KeePassium, aller dans Settings > Préremplir les mots de passe, activer l’option de pré-remplissage des mots de passe, ce qui permettra de copier les OTP dans le presse-papiers. • Utiliser KeePassium en auto-remplissage : sur un site web, dans le champ d’authentification, utiliser les mots de passe enregistrés dans KeePassium (via l’outil d’auto-remplissage). Sous Android : • Télécharger Keepass2Android : Site officiel [https://github.com/PhilippC/keepass2android], Google Play Store [https://play.google.com/store/apps/details?id=keepass2android.keepass2android&hl=en-US]. • Définir Keepass2Android comme service de remplissage automatique (exemple sur Samsung Galaxy A72 : Paramètres > Gestion globale > Mots de passe > Service de remplissage auto > Keepass2Android). • Créer une base de données : ouvrir l’application, créer une base de données (choisir un emplacement pour le fichier, définir un mot de passe sécurisé). • Confirmer les modifications : déverrouiller la base de données avec le mot de passe (taper les trois derniers caractères pour confirmation ou utiliser la biométrie si disponible). • Créer un groupe : par exemple un groupe « Meta » pour regrouper les informations de comptes Facebook, Instagram, etc. • Créer une entrée : renseigner les champs (nom de l’entrée, nom d’utilisateur, URL de connexion, mot de passe, etc.). Sauvegarder le fichier KeePass (.kdbx) • Stockage : contient les mots de passe, identifiants, OTP (TOTP) de l’utilisateur. • Sauvegarde : possibilité de stocker la base sur iCloud Drive, Google Drive, NAS, clé USB sécurisée… (penser à chiffrer avec un mot de passe fort). VI. Intégrer un OTP (via le champ OTP avec plugins) • OTP : Code temporaire unique utilisé pour vérifier l’identité dans le cadre d’une authentification à deux facteurs (2FA). Sous iOS : • KeePassium : configurer un OTP via un QR code ou une saisie manuelle de la clé secrète. • Restrictions Apple : l’intégration n’est pas complète avec l’écosystème d’auto-remplissage des codes OTP (limitation iOS). • Solution alternative : copier-coller manuellement l’OTP depuis KeePassium, ou utiliser les applications Apple Passwords / Microsoft Authenticator pour les codes. Sous Android : • Ajouter une entrée OTP dans Keepass2Android : dans la base de données, créer une nouvelle entrée, renseigner la section TOTP avec la clé secrète en base32, l’algorithme (SHA-1) et l’intervalle (30 secondes). • Générer un OTP : sélectionner l’entrée contenant l’OTP, appuyer sur l’icône horloge pour copier le code dans le presse-papiers. • Scanner un QR Code : pour configurer un TOTP, scanner le QR code OTP fourni par le service lors de l'activation. VII. Exemples génériques de fonctionnement d’un OTP • OTP : Code temporaire généré automatiquement, qui renforce la sécurité lors d’une connexion ou d’une opération sensible. • Mise en place d’un OTP : Activer l’authentification 2FA dans les paramètres de sécurité d’un service, puis choisir une application d'authentification pour générer les codes. • Activer l’authentification 2FA : Dans les paramètres de sécurité du service, activer l’option et choisir une application d'authentification. • QR Code et clé secrète : Scanner le QR code fourni ou copier la clé secrète dans le presse-papiers. • Ajouter la clé secrète dans l’application OTP : Ouvrir l’application d'authentification, scanner le QR code ou entrer manuellement la clé secrète, sauvegarder ; un code à 6 chiffres est alors généré automatiquement en continu. • Connexion au site/service : Se connecter avec le nom d’utilisateur, le mot de passe, puis entrer le code OTP fourni par l’application d'authentification. • Vérification de validité de l’OTP : Le site/service vérifie le code OTP. En cas d’erreur ou d’expiration du code (valable généralement 30 sec), renouveler l’opération avec le nouveau code OTP à l’écran. VIII. Cas d'usages • Perte d’accès aux comptes en ligne : Récupération de mot de passe oublié, ajout de contacts de secours. 1. Mot de passe oublié Sous Windows : • Code de récupération (compte Microsoft) : se connecter au compte Microsoft > Tableau de bord Sécurité > Gérer la façon de se connecter > Code de récupération > Générer un nouveau code. • Imprimer le code de récupération obtenu et le conserver en lieu sûr (gestionnaire de mots de passe, base de données cloud, etc.). • Sur la page de connexion au compte Microsoft, entrer le nom d’utilisateur puis utiliser l’option « Code de récupération » et saisir le code imprimé. • Pour retrouver/télécharger un code de récupération existant : se connecter au compte Microsoft, créer un nouveau code de récupération (si l’ancien est perdu), ou lors d’un oubli de mot de passe, choisir « autres moyens de connexion » puis « Code de récupération du compte ». Sur iPhone : • Réglages > [Votre nom] > Mot de passe et sécurité > Clé de secours > Activer > Continuer. Générer une clé de secours, la noter et la conserver en lieu sûr, puis confirmer la clé de secours sur l'appareil. Sur Mac : • Menu Pomme > Réglages Système > [Votre nom (Apple ID)] > Mot de passe et sécurité > Clé de secours > Activer. Suivre les étapes : saisir le mot de passe Mac, copier la clé de secours générée, puis confirmer pour l’activer. • Utiliser la clé de secours sur le compte Apple : en cas de mise à jour logicielle de l’appareil ou d’identification à deux facteurs, on peut vous demander la clé de secours. L’activer désactive le processus de récupération standard du compte Apple [https://support.apple.com/fr-fr/118574]. • Modifier le mot de passe du compte Apple : nécessite un appareil de confiance, le code d’accès de celui-ci et un numéro de téléphone de confiance. • En cas de perte d’accès au compte Apple : utiliser la clé de secours et le code de vérification envoyé au numéro de téléphone de confiance. Conseils (compte Apple) : • Toute modification des informations du compte Apple nécessitera la clé de secours. • Pour désactiver la clé de secours, suivre les étapes sur un appareil de confiance. • Configurez vos contacts de récupération sur iCloud et conservez la clé de secours en lieu sûr. • La clé de récupération Apple est une couche de sécurité supplémentaire qui nécessite une gestion rigoureuse (à conserver en lieu sûr absolument). Sous Android : • Codes de sauvegarde (compte Google) : sur votre téléphone Android, ouvrir l’application Paramètres > Google > Gérer votre compte Google > Sécurité > Validation en deux étapes > Codes de sauvegarde > Générer/Afficher les codes. • Obtenir des codes de sauvegarde : vous pouvez créer un nouvel ensemble de codes, supprimer les anciens, télécharger ou imprimer les codes de sauvegarde fournis. Conseils (comptes Google) : • En cas de perte des codes, en générer de nouveaux et stocker la liste en lieu sûr (et idéalement en garder une copie imprimée). • N’utiliser les codes de sauvegarde qu’en dernier recours (si la 2FA est activée et le téléphone inaccessible). • Ne jamais partager vos codes de secours. • Pour KeePass Android : pensez à sauvegarder vos bases de données KeePass sur le cloud et à conserver une copie hors-ligne (ainsi que la feuille de secours du mot de passe maître). Retrouver un code de sauvegarde perdu : • Si les codes étaient téléchargés sur votre appareil, chercher un fichier nommé « Backup-codes-googleXXXX.txt » (par exemple, avec votre nom d’utilisateur). • Sur la page de connexion Google, choisir « Essayer autrement » puis « Code de sauvegarde à 8 chiffres », et entrer un code non utilisé de votre liste de codes de secours. 2.
